### 1. Functional Requirements (FR)
* **FR-1: User Classification:** The system must capture whether a reporter is a student, indigene, or resident.
* **FR-2: Location Mapping:** The interface must require users to select their settlement (Akungba, Iwaro, Etioro, or Ayegunle).
* **FR-3: Incident Categorization:** Users must choose from pre-defined issues: Security Threat, Rent Dispute, Price Exploitation, or Infrastructure Deficits.
* **FR-4: Anonymous Mode Toggle:** The system must allow a user to hide their personal identity upon submission for security reasons.
* **FR-5: Multi-media Attachment:** The platform must support local file uploads to attach photo evidence of incidents.
* **FR-6: Status Tracking Pipeline:** The dashboard must display real-time complaint status records tagged as Pending or Resolved.

### 2. Non-Functional Requirements (NFR)
* **NFR-1: Security and Confidentiality:** High-security protection algorithms must obscure details of anonymous whistleblowers.
* **NFR-2: Performance and Speed:** The landing UI must load within 3 seconds under standard local 3G wireless connectivity conditions.
* **NFR-3: Device Accessibility:** The frontend architecture must remain completely responsive across diverse mobile devices.
